6

Communication

HTTP (Polling)


  • half-duplex
  • client continually sends HTTP requests to the server to ask for new data
  • server sends HTTP response with message (or command to terminate the request)

  1. many connections have to be handled for each client
  2. each HTTP header contains redundant data
7

Communication

Web Sockets Protocol (Push)


  • full-duplex
  • once a Web Socket connection has been established client and server can push data to each other

  1. one connection per client
  2. no data overhead
